mcp-rtc host — phone capabilities exposed as MCP tools: take_photo, get_location, read_clipboard, get_orientation

To call these from a stdio MCP client (Claude Code, Codex, Cursor, Claude Desktop), install the bridge once and connect with this page's site id (the URL fragment, e.g. ph-abc123):

# Claude Code:  claude mcp add bridge -- npx -y @nevescloud/mcp-rtc-bridge
# Codex CLI:    codex mcp add bridge -- npx -y @nevescloud/mcp-rtc-bridge
# Cursor / Desktop / generic — add @nevescloud/mcp-rtc-bridge to mcpServers config

Then: connect({ siteId: "<id-after-#-in-this-URL>" }) and call peer_take_photo(), peer_get_location(), etc. The phone prompts the user once per capability.

Your phone is now an MCP server.

It exposes the tools below to any Claude (or other AI) on the other end of mcp-rtc. Permissions are requested the first time each tool is called.

📷 take_photo — rear camera, returns a JPEG on demand
📍 get_location — GPS coords with accuracy on demand
📋 read_clipboard — text from system clipboard on demand
🧭 get_orientation — device tilt (alpha/beta/gamma) on demand
