Turn any browser tab into an MCP server.

Peer-to-peer over WebMCP and WebRTC. Callable by Claude Code, Cursor, Claude.ai, or any MCP client.

Tab The MCP server. @nevescloud/mcp-rtc → mcp-rtc WebRTC data channel, peer-to-peer. →
Claude Code via stdio bridge npx @nevescloud/mcp-rtc-bridge Claude.ai via bridge tab + WebMCP Chrome 146+ with consumer

Try it

Call a browser tab as a tool from Claude.

  1. Open the host page. The tab becomes an MCP server with one tool, get_greeting. Note the #hi-XXXXXX in the URL. Open the host page →
  2. Connect Claude to that site id:

    Open the bridge page with your site id in the fragment, in a Chrome 146+ tab where a WebMCP consumer (e.g. the Anthropic Claude extension) is attached. Ask Claude "call get_greeting". The call routes peer-to-peer over WebRTC; the result comes back through WebMCP.

    Open the bridge page →

    Requires: Chrome 146+ with a WebMCP consumer (e.g. the Anthropic Claude extension).

    Install the bridge once:

    claude mcp add mcp-rtc-bridge -- npx -y @nevescloud/mcp-rtc-bridge

    Then copy the "Paste in Claude Code" prompt from the host page and paste it. Claude calls connect with the site id, lists the remote tools, then asks what to do.

    Codex CLI is the same command with codex mcp add …. Cursor, Claude Desktop, and other stdio MCP clients accept npx -y @nevescloud/mcp-rtc-bridge in their mcpServers config UI.

On your phone

Phone as Claude-callable tools.

Scan with your phone's camera, or open neves.cloud/p/ on the phone. The page registers take_photo, get_location, read_clipboard, get_orientation — each prompts for permission lazily on the first call.

The phone shows a Paste in Claude Code prompt with its site id. Install the bridge once with claude mcp add mcp-rtc-bridge -- npx -y @nevescloud/mcp-rtc-bridge, paste the prompt, and your laptop's Claude reaches the phone.
